### Hexlet tests and linter status:
[![Actions Status](https://github.com/gribanovaleksey17/qa-engineer-project-84/actions/workflows/hexlet-check.yml/badge.svg)](https://github.com/gribanovaleksey17/qa-engineer-project-84/actions)

Проект посвящен тестированию SPA интернет-магазина Hexlet Products Store.
Приложение позволяет пользователю просматривать каталог товаров, применять фильтры, выполнять поиск, добавлять товары в корзину и оформлять заказ.

В ходе работы были выполнены ключевые этапы тестирования, начиная с анализа требований и заканчивая регрессионным тестированием новой версии продукта.

Цели тестирования:
- проверить корректность работы основного функционала интернет-магазина
- убедиться, что пользовательские сценарии работают стабильно
- выявить дефекты в функциональности, локализации и UI
- провести регрессионное тестирование обновленной версии приложения

В рамках проекта выполнено:
- тест-анализ требований и их классификация
- (functional / non-functional / implicit)
- проектирование тест-кейсов на основе требований
- функциональное тестирование web-приложения
- фиксация результатов тестирования (Pass/Fail)
- оформление баг-репортов с описанием шагов воспроизведения и окружения
- проведение регрессионного тестирования версии v2

Артефакты проекта:
- about.yml — информация обо мне
- requirements.yml — требования, распределенные по категориям
- test-cases.yml — тест-кейсы на функциональные сценарии
- testing-report.yml — отчет о прохождении тест-кейсов
- bugreports.yml — баг-репорты по найденным дефектам
- regress-report.yml — результаты регрессионного тестирования версии v2

Во время тестирования использовались практики, приближенные к production-подходу:
- проверки UI/UX поведения интерфейса
- анализ запросов и ошибок через Chrome DevTools (Network/Console)
- проверка локализации интерфейса (RU/EN)
- проверка сохранения состояния корзины в сессии браузера
- анализ пользовательских сценариев: каталог → товар → корзина → оформление заказа
